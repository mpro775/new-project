# وثائق مشروع زيتونة (Zaytuna POS System)

## نظرة عامة
هذا المجلد يحتوي على جميع الوثائق التقنية والتجارية لمشروع نظام نقاط البيع زيتونة.

## هيكل الوثائق

```
docs/
├── README.md                              # هذا الملف - نظرة عامة على الوثائق
├── technical-documentation-guide.md       # دليل الوثائق التقنية المطلوبة
│
├── 01-planning/                           # وثائق التخطيط (قبل التنفيذ)
│   ├── business/                          # الوثائق التجارية
│   ├── technical/                         # الوثائق التقنية
│   └── README.md
│
├── 02-development/                        # وثائق التطوير (أثناء التنفيذ)
│   ├── backend/                           # وثائق الباك إند
│   ├── frontend/                          # وثائق الفرونت إند
│   ├── api/                              # وثائق API
│   └── README.md
│
├── 03-operations/                         # وثائق التشغيل (بعد التنفيذ)
│   ├── deployment/                        # دليل النشر
│   ├── monitoring/                        # دليل المراقبة
│   ├── user-guides/                       # أدلة المستخدمين
│   └── README.md
│
├── 04-maintenance/                        # وثائق الصيانة والدعم
│   ├── troubleshooting/                   # حل المشاكل
│   ├── updates/                           # التحديثات والترقيات
│   └── README.md
│
├── assets/                                # الملفات المساعدة
│   ├── diagrams/                          # المخططات والرسوم البيانية
│   ├── screenshots/                       # لقطات الشاشة

│
└── templates/                             # قوالب الوثائق
    ├── project-templates/                 # قوالب المشاريع
    └── document-templates/                # قوالب الوثائق
```

## كيفية استخدام الوثائق

### للمطورين الجدد:
1. ابدأ بقراءة `technical-documentation-guide.md`
2. اقرأ وثائق التخطيط في `01-planning/`
3. راجع وثائق التطوير في `02-development/`

### لفريق العمليات:
1. راجع وثائق التشغيل في `03-operations/`
2. اقرأ أدلة النشر والمراقبة

### للمستخدمين النهائيين:
1. راجع أدلة المستخدمين في `03-operations/user-guides/`

## المسؤوليات

- **مدير المشروع**: مراجعة واعتماد وثائق التخطيط
- **مهندس الحلول**: إعداد وثائق البنية التقنية
- **المطورون**: صيانة وثائق التطوير والAPI
- **فريق العمليات**: إعداد وثائق النشر والتشغيل
